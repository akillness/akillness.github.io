When learning complex technologies, it is always essential to build a solid foundational base.
The following is a breakdown of the technologies and tools to invest in while working with or transitioning to **CloudNative**.

- 👉 𝑶𝑺 𝑭𝒖𝒏𝒅𝒂𝒎𝒆𝒏𝒕𝒂𝒍𝒔 - Learning this improves your understanding of how software that powers the cloud actually works. Popular choices are Linux and Ubuntu. Practicing and understanding these is essential for gaining experience in server and network administration.

- 👉 𝑵𝒆𝒕𝒘𝒐𝒓𝒌𝒊𝒏𝒈 𝑩𝒂𝒔𝒊𝒄𝒔  - Whether building or designing systems, networking is a critical factor that helps you understand CIDR, IP subnets, firewalls, and more, as well as virtualization and container networking.

- 👉 𝑪𝒍𝒐𝒖𝒅 𝑺𝒆𝒓𝒗𝒊𝒄𝒆 𝑷𝒓𝒐𝒗𝒊𝒅𝒆𝒓𝒔  - Understanding popular clouds and their services can also help with cloud architecture, applications, hosting, migration, and more for any project or client #consulting service.

- 👉 𝑺𝒆𝒄𝒖𝒓𝒊𝒕𝒚 𝑰𝒏𝒕𝒆𝒓𝒏𝒂𝒍𝒔  - Security plays a pivotal role in managing the information security, assurance, and risk-based processes essential for addressing the latest threats in cloud-native businesses.

- 👉 𝑪𝒐𝒏𝒕𝒂𝒊𝒏𝒆𝒓𝒔 𝑫𝒆𝒆𝒑-𝑫𝒊𝒗𝒆  - Migrating from legacy/monolithic systems to #Microservices models requires a thorough understanding of container architecture, capabilities, and drawbacks — hence docker, Kubernetes, and other container technologies.

- 👉 𝑰𝒏𝒇𝒓𝒂𝒔𝒕𝒓𝒖𝒄𝒕𝒖𝒓𝒆 𝒂𝒔 𝑪𝒐𝒅𝒆 (𝑰𝒂𝑪)  - Managing servers at scale requires transitioning from manual processes to more automated ones that can declaratively manage infrastructure, provide version control, and reduce possible human errors.

- 👉 𝑪𝒐𝒏𝒕𝒊𝒏𝒖𝒐𝒖𝒔 𝑰𝒏𝒕𝒆𝒈𝒓𝒂𝒕𝒊𝒐𝒏/𝑪𝒐𝒏𝒕𝒊𝒏𝒖𝒐𝒖𝒔 𝑫𝒆𝒍𝒊𝒗𝒆𝒓𝒚 (𝑫𝒆𝒑𝒍𝒐𝒚𝒎𝒆𝒏𝒕) 𝒂𝒑𝒑𝒓𝒐𝒂𝒄𝒉  - Learning to use tools like GitHub, Bitbucket, Flux, ArgoCD, and Jenkins can help modernize the software development lifecycle. It catches errors earlier and improves overall quality in modern cloud-based applications compared to manual processes.

- 👉 𝑶𝒃𝒔𝒆𝒓𝒗𝒂𝒃𝒊𝒍𝒊𝒕𝒚 - Learning to use tools like Elastic Stack, Grafana, Splunk, and Prometheus can help improve system/application observability by resolving early issues, safely experimenting with and implementing optimizations, and managing business risk. This helps maintain SLAs, SLOs, and SLIs for applications and systems built on the above technology stack.
